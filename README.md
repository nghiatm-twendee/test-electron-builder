# Txtpad

A lightweight desktop plain text editor built with Electron. A port of the [Txtpad PWA](../vite-pwa-txt-editor) to explore Electron's architecture, IPC patterns, and packaging.

---

## Purpose

This is a learning project with three goals:

1. **Explore Electron** — understand the main/preload/renderer process model, IPC, contextBridge, BrowserWindow, and electron-builder packaging.
2. **Learn Claude Code** — the project is built collaboratively with [Claude Code](https://claude.ai/code) as an AI pair-programmer, exploring how to use it effectively for desktop app development.
3. **Port a familiar app** — bring the Txtpad PWA to the desktop so the focus stays on Electron and Claude Code concepts rather than building features from scratch.

---

## Features

- **Open, create, save, and close `.txt` files** using Node.js `fs` via the preload/IPC bridge — no browser File System Access API required
- **Multiple files open at once** with a collapsible sidebar for switching between them
- **Unsaved-changes indicator** — a dot badge marks dirty files; closing prompts for confirmation
- **Keyboard shortcuts** for every action (no browser conflicts — `Ctrl+W`, `Ctrl+N` are available in Electron):

  | Action        | Shortcut               |
  |---------------|------------------------|
  | New file      | `Ctrl+N` / `⌘N`        |
  | Open file     | `Ctrl+O` / `⌘O`        |
  | Save          | `Ctrl+S` / `⌘S`        |
  | Save As       | `Ctrl+Shift+S` / `⌘⇧S` |
  | Close         | `Ctrl+W` / `⌘W`        |
  | Previous file | `Alt+↑`                |
  | Next file     | `Alt+↓`                |

- **Dark / light / system theme** toggle persisted to `localStorage`
- **Auto-focus textarea** after every file action

## Getting started

Node.js >= 23.0.0 is required.

```bash
# First-time setup — creates the renderer package, integrates it, installs deps
npm run init

# Start in development mode with hot-reload
npm start

# Type-check all packages
npm run typecheck

# Build all workspace packages
npm run build

# Compile to a distributable executable
npm run compile

# Compile without asar (useful for debugging the compiled app)
npm run compile -- --dir -c.asar=false

# Run end-to-end tests (requires a compiled app in dist/)
npm run test
```

## Architecture

```
renderer  ←→  preload  ←→  main
```

- **`packages/main`** — Electron main process. Features are implemented as `AppModule` classes registered via `ModuleRunner` (window management, auto-updater, security policies, single-instance lock).
- **`packages/preload`** — Preload script. Exports are automatically exposed to the renderer via `contextBridge.exposeInMainWorld`. Keys are base64-encoded. The renderer imports from `@app/preload` as if it were a normal ESM module — the build system rewrites the calls transparently.
- **`packages/renderer`** — The React UI. Cannot use Node.js APIs directly; all native access goes through the preload bridge.

See the [vite-electron-builder](https://github.com/cawa-93/vite-electron-builder) boilerplate for deeper documentation on the build system, IPC patterns, environment variables, and packaging.
